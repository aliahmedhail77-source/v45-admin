# ملاحظات V45 Stage 13.8
## Reports, Logs, Wallet Review, Message Templates Cleanup

تم اعتماد هذه المرحلة كتنظيف وتحسين آمن على نسخة Stage 13.7.3 دون تغيير بنية المشروع أو منطق بيع الكروت المستقر.

## 1) التقارير
- تحويل واجهة السجل إلى عنوان: السجلات.
- جعل تقرير العمليات هو التقرير الأساسي بدل فكرة تقرير إجراءات العميل.
- تقليل النصوص والتفاصيل داخل تقرير الدفتر حتى يكون حجم PDF أصغر.
- كشف حساب العميل أصبح باسم: تقرير العمليات.
- إضافة أعمدة الرصيد قبل وبعد داخل كشف حساب الدفتر.
- منع زر تقرير PDF منفصل لكل عملية من شاشة السجل لتقليل حجم الملفات وتكرار التقارير.

## 2) السجلات
- إخفاء سجلات الطابور التقنية من واجهة المستخدم.
- لا تظهر رسائل: في الطابور، قيد معالجة الطابور، تمت قراءة الطابور.
- السجل الظاهر مخصص للعمليات المفيدة فقط: سداد، سلفة، بيع، تعبئة، مراجعة، فشل مهم.
- الصفحة الرئيسية والسجلات والتقارير الدورية تستخدم السجلات الظاهرة بعد الفلترة.

## 3) طابور الرسائل SMS Queue
- تم إيقاف إنشاء سجل ظاهر عند دخول الرسالة للطابور.
- العملية الناتجة فقط هي التي تظهر بعد المعالجة.
- إذا حدث فشل حقيقي أثناء المعالجة يتم إنشاء سجل فشل مفيد بدل سجل قراءة داخلي.

## 4) المحافظ والسداد بدون رقم
- تحسين نص المراجعة عندما تصل عملية سداد باسم بدون رقم صحيح.
- توجيه المستخدم إلى إضافة الاسم للمحافظ الموثوقة مع إرفاق الرقم الصحيح قبل الاعتماد.
- الإشعار يظل ضمن عمليات المراجعة حتى تتم معالجته.

## 5) آخر القيود
- آخر القيود في الدفتر تعرض أول 5 قيود فقط.
- زر المزيد يعرض 20 قيدًا إضافيًا كل مرة.
- هذا يقلل تحميل الواجهة عند وجود قيود كثيرة.

## 6) واجهة إضافة الزبون
- تحسين واجهة إضافة/تعديل الزبون بتقسيم واضح:
  - بيانات الزبون.
  - إعدادات السلفة والرصيد.
  - ملاحظات داخلية.
  - طريقة التعامل.
- إضافة تعليمات وأمثلة تحت الحقول المهمة.

## 7) تعديل الرسائل
- إضافة قسم جديد داخل مركز تعديل الرسائل:
  - رسائل القيود والسداد.
  - رسالة تعبئة رصيد نقاط البيع.
- المتغيرات الجديدة للسداد:
  - {customerName}
  - {amount}
  - {debtBefore}
  - {debtAfter}
  - {creditBefore}
  - {creditAfter}
  - {network}
  - {signature}
- المتغيرات الجديدة لتعبئة نقطة البيع:
  - {amount}
  - {oldRemaining}
  - {remaining}
  - {network}
  - {signature}

## 8) بصمة الرسائل
- رسائل السداد والتعبئة تستخدم بصمة الرسائل عبر اسم الشبكة افتراضيًا.
- إذا غيّر المستخدم اسم الشبكة من الإعدادات تتغير البصمة تلقائيًا.

## 9) إدارة البنوك والمحافظ
- تعديل مسميات الواجهة إلى: إدارة البنوك والمحافظ والأسماء الموثوقة.
- تحسين زر الدخول إلى الإدارة بأيقونة واضحة.

## ملاحظات أمان
- لم يتم حذف بيانات مخزنة.
- لم يتم تغيير بنية قاعدة البيانات.
- لم يتم تغيير مسار البيع التلقائي الأساسي للكروت.
- لا يزال السداد باسم بدون رقم يحتاج مراجعة إذا لم يتم ربطه برقم صحيح.

## 10) زر الرجوع الداخلي
- تم حذف أزرار الرجوع النصية من النوافذ/الشاشات التي ظهر فيها زر رجوع داخلي.
- الاعتماد الآن على زر الرجوع الخاص بالهاتف، مع بقاء أزرار الحفظ والإلغاء داخل الحوارات المهمة.
